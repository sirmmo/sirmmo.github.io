# Impeccable Design Context

This file is the source of truth for design decisions on this project. Read it before any design work.

## Design Context

### Users

Three concentric audiences, all visiting in browser-on-a-laptop mode. Not a personal canonical record — the site needs to land well with outsiders, not just file what Marco does.

1. **Freelance & consulting leads** — deciding whether to commission Marco for data, GIS, Angular, or Python work. Scanning for credibility and concrete work.
2. **Open-source collaborators** — people from the GIS / OpenStreetMap / history / tabletop scenes considering joining Open History Map, Open Fantasy Maps, or Just Play Bologna.
3. **Peers & curious technologists** — engineers, researchers, and academics looking around to see what Marco is working on.

### Brand Personality

**Curious, polymathic, opinionated.** Marco self-describes as "a strange engineer who loves AI (when it was still called Machine Learning), Philosophy, Sociology, Theology... urbanistics and architecture." The site should let that breadth show without being chaotic — restless intelligence that has chosen a frame.

Voice: first-person, plain-spoken, willing to have a position. Not PR copy. Not a buzzword skills list.

### Aesthetic Direction

**Cartographic / atlas.** The site is a map of a person — and that is not a stretched metaphor, since Marco's two flagship projects are literally maps (Open History Map, Open Fantasy Maps). Lean into:

- Typographic legends, coordinates, scale bars, considered margins
- Paper / ink / cinnabar / woodblock-green palette territory in OKLCH (no pure black, no pure white)
- Hand-drawn map textures used sparingly and as content, not decoration
- Asymmetric layout with strong typographic hierarchy and a distinctive display/body pairing (avoid the reflex font list — see skill guidance)
- Both light and dark modes designed with equal care; theme respects `prefers-color-scheme` with a manual toggle (the existing sun/moon control stays, but neither mode is a fallback)

**Anti-references — what this should not look like:**

- The current site itself (dark Tailwind dev-portfolio template, Font Awesome icons everywhere, project card grid)
- Corporate consultancy / agency stock-photo vibes ("we help businesses transform")
- AI-startup chrome (cyan-on-dark, purple-blue gradients, glassmorphism, glowing accents)
- Aggressive maximalism — maps can be quiet

### Design Principles

1. **The map is the metaphor, not the wallpaper.** Cartographic devices (legends, scales, projections, coordinates) should organize the page, not sit on it as ornament. If a flourish doesn't help someone navigate, cut it.
2. **Show breadth without sprawl.** Marco's range is the story, but a polymath site that *looks* polymathic reads as scattered. Use one rigorous typographic and spatial system across mapping projects, freelance work, teaching, hobbies, and activism.
3. **Earn the dark mode.** The current dark default is the AI-portfolio reflex. Both themes must be designed with intent.
4. **No identical card grid for projects.** That's the explicit anti-reference. Projects should sit on the page like points on an atlas plate, not as repeating rectangles.
5. **Italian, in Bologna, plainly.** Place is part of the brand. Don't sand it off for a generic-international feel.

## Technical Constraints

- Hugo extended static site, hand-rolled `layouts/` (no theme).
- Content is data-driven from `data/*.json` rendered by matching `layouts/partials/*.html`. **Edit the JSON for content, the partial for layout.**
- `static/css/main.min.css` is a prebuilt Tailwind bundle checked in directly — there is no build pipeline. New design work either edits this file, replaces it, or moves to plain CSS in `static/css/` linked from `layouts/partials/head.html`.
- Font Awesome and Academicons are CDN-loaded; if the new direction does not need them, drop the links rather than leaving them in.
- Site is single-page (`layouts/index.html`); no router, no client framework.
